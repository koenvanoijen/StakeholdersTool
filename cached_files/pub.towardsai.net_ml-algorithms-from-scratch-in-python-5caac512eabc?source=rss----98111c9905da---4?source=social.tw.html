Linear Regression

Gradient Descent is the process of minimizing a function by following the gradients of the cost function. This involves knowing the form of the cost as well as the derivative so that from a given point you know the gradient and can move in that direction, e.g. downhill towards the minimum value.

We are following Gradient descent to find B0 and B1 which would minimize the loss function(Mean Squared Error) in this case.
Initialisations:
B0=B1=0 , Learning rate(lr)=0.01, #Epochs=500

Pseudocode:
1. Input Xs and Ys are fed to the system with init values
2. Prediction of Y is made with B0*X+B1
3. Error Values are returned for coeffient correction
4. B0=B0-lr*Error and B1=B1-lr*X*Error
5. After a point lets say 300 epochs, error function would start converging i.e no change and that would be a point of early stopping.

K-Means:

Optimum K is calculated by elbow Method

In the Elbow method, we are actually varying the number of clusters ( K ) from 1–10. For each value of K, we are calculating WCSS ( Within-Cluster Sum of Square ). WCSS is the sum of squared distance between each point and the centroid in a cluster. When we plot the WCSS with the K value, the plot looks like an Elbow. As the number of clusters increases, the WCSS value will start to decrease. WCSS value is largest when K = 1. When we analyze the graph we can see that the graph will rapidly change at a point and thus creating an elbow shape. From this point, the graph starts to move almost parallel to the X-axis. The K value corresponding to this point is the optimal K value or an optimal number of clusters.

We can pick the number of clusters by varying it in a range of possible values and compute the silhouette score and the distortion at every iteration. The optimal number is the one that maximizes the first and minimizes the second.
The two methods to measure the cluster quality are described below:

Inertia: Intuitively, inertia tells how far away the points within a cluster are. Therefore, a small of inertia is aimed for. The range of inertia’s value starts from zero and goes up.

Silhouette score: The silhouette score tells how far away the datapoints in one cluster are, from the datapoints in another cluster. The range of silhouette scores is from -1 to 1. The score should be closer to 1 than -1.
The elbow method looks at the percentage of variance explained as a function of the number of clusters: One should choose a number of clusters so that adding another cluster doesn’t give much better modeling of the data. More precisely, if one plots the percentage of variance explained by the clusters against the number of clusters, the first clusters will add much information (explain a lot of variances), but at some point the marginal gain will drop, giving an angle in the graph. The number of clusters is chosen at this point, hence the “elbow criterion”.

Kmeans separates the documents(NLP) into disjoint clusters. the assumption is that each cluster is attributed to a single topic. However, descriptions may in reality be characterized by a “mixture” of topics. For example, let’s take an article that deals with the hearing that Zuckerberg had in front of the congress: you’ll obviously have different topics rising based on the keywords: Privacy, Technology, Facebook app, data, etc. LDA helps deal with this problem.

Input is n data points with k clusters, small tolerance limit such as 0.01 after for early stopping and n_iterations as 500. The output is a set of ‘k’ cluster centroids and labeling of the dataset that maps each of the data points to a unique cluster.

Step 1. Randomly pick k data points as our initial Centroids.
Step 2. Find the distance (Euclidean distance for our purpose) between each datapoints in our training set with the k centroids.
Step 3. Now assign each data point to the closest centroid according to the distance found.
Step 4. Update centroid location by taking the average of the points in each cluster group.
Step 5. Repeat Steps 2 to 4 till our centroids don’t change.

Perceptron

Perceptron receives input signals from examples of training data that we weigh and combined in a linear equation called the activation. It is closely related to linear regression and logistic regression that make predictions in a similar way (e.g. a weighted sum of inputs). The weights of the Perceptron algorithm must be estimated from your training data using stochastic gradient descent.

The weights and biases are initialized as 0 and learned based on error from predicted vs output value by activation = sum(weight_i * x_i) + bias.
The weight matrix is equal to input matrix or num_neurons

Error Backpropagation

The first step is to calculate the error for each output neuron, this will give us our error signal (input) to propagate backward through the network. The error for a given neuron can be calculated as follows

error = (output — expected) * transfer_derivative(output)

Where expected is the expected output value for the neuron, the output is the output value for the neuron and transfer_derivative() calculates the slope of the neuron’s output value
Vanishing gradient means that weights in layers close to the input layer are not updated in response to errors calculated on the training dataset. A problem with training networks with many layers (e.g. deep neural networks) is that the gradient diminishes dramatically as it is propagated backward through the network. The error may be so small by the time it reaches layers close to the input of the model that it may have very little effect. As such, this problem is referred to as the “vanishing gradients” problem. Vanishing gradients is a particular problem with recurrent neural networks as the update of the network involves unrolling the network for each input time step, in effect creating a very deep network that requires weight updates.
Solutions: pre-training, better random initial scaling, better optimization methods, specific architectures, orthogonal initialization, etc.

TF-IDF (Term Frequency-Inverse Document Frequency)

Term Frequency: It is the ratio of the number of times the word appears in a document compared to the total number of words in that document.
tf(t,d) = count of t in d / number of words in d
For corpus- “Ravi is here. Rohan is there”,
tf(Ravi)=1/6

Disadvantages of TF-IDF:
- TF-IDF is based on the bag-of-words (BoW) model, therefore it does not capture the position in the text, semantics, co-occurrences in different documents, etc.
- Cannot capture semantics (e.g. as compared to topic models, word embeddings)

Word embedding represents the vector representation of words in a constrained space e.g 100 Dimensional space as opposed to Millions of possible worlds (dimensions) in the dictionary. 3.Combine individual samples for a uniform weight matrix, e.g 12 weights with 4 dimension each. Use padding for small sized reviews.
4.Pass the matrix through a neuron, calculate the error against the labelled data(Fake problem) and backpropagate for updating the input weights.

Recurrent Neural Network(RNN)- For sequence problems such as next word prediction, time series, speech recognition, etc where output is dependent on the previous sequence.

Steps for RNN from scratch:
1. Data preparation with the sequence as input e.g trailing 50 days of stock prices
2. Architecture creation for RNN: Inputs are LR, #Epochs, hidden layer, input layer, and output layer similar to DNN with additional parameters as BPPT(Backpropagation through time), Min_Clip and Max_Clip for vanishing/exploding gradient. The core difference in BPTT versus backprop is that the backpropagation step is done for all the time steps in the RNN layer. So if our sequence length is 50, we will backpropagate for all the timesteps previous to the current timestep.
In addition to the input and output weight matrix, we would have an additional matrix for shared weights in the RNN layer.
3. Train the model: Forward Pass, Calculate loss, Start training, Backpropagate error and update weights
4. Generate Predictions

Gradient clipping is a technique used to cope with the exploding gradient problem sometimes encountered when performing backpropagation. By capping the maximum value for the gradient, this phenomenon is controlled in practice.
Gradient Update rule, New Weight=weight-LR*Gradient,
If Gradient is small, no message is reached to earlier layers.
If Gradient is large, the Network wouldn't converge and show NaN errors.

LSTM: They have Gates to remove unwanted noise.
It has a cell state and Gates: Forget gate, Input Gate and Output Gate.
Gates contain sigmoid activation i.e update or forget[1,0] unlike Tanh[-1,1].
Forget Gate: Information from the previous hidden state and the current state is passed through the sigmoid function. Closer to 0 means forget and closer to 1 means keep.
Input Gate: Previous hidden and current input is passed through the Tanh function. Tanh output and Sigmoid output are multiplied for the next state.
Output Gate: Decides what the next hidden state should be.
